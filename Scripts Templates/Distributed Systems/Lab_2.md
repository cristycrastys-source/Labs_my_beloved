## Теоретические основы

### HTTP (HyperText Transfer Protocol)
Протокол прикладного уровня для передачи данных. Клиент отправляет HTTP-запрос, сервер отвечает HTTP-ответом.

**Запрос:** метод (GET, POST, PUT, DELETE), URI, заголовки, опциональное тело.
**Ответ:** код состояния (200 OK, 404 Not Found, 301 Moved Permanently), заголовки, тело ответа.

### Telnet и Curl
- **telnet** — утилита для взаимодействия с удаленными хостами по заданному порту. Позволяет отправлять "сырые" HTTP-запросы.
- **curl** — инструмент для отправки запросов по различным протоколам, автоматически обрабатывает редиректы.

### Nginx
Высокопроизводительный веб-сервер. Может выполнять две ключевые роли:
- **Веб-сервер** — напрямую отдает статические файлы.
- **Обратный прокси (Reverse Proxy)** — принимает запросы от клиентов и перенаправляет их на другие серверы (Flask-приложение).

### REST и RESTful API
REST (Representational State Transfer) — архитектурный стиль для создания распределенных систем.

**Ключевые принципы:**
- Клиент-серверная архитектура
- Отсутствие состояния (Stateless)
- Использование стандартных методов HTTP: GET (получить), POST (создать), PUT/PATCH (обновить), DELETE (удалить)
