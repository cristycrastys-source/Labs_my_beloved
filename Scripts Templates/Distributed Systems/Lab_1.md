## Теоретические сведения

### RPC (Remote Procedure Call)
Технология, позволяющая программе вызывать процедуру в другом адресном пространстве (на другом компьютере в сети), как если бы это был локальный вызов. Упрощает разработку распределенных приложений, скрывая детали сетевого взаимодействия.

### gRPC
Современный высокопроизводительный фреймворк RPC с открытым исходным кодом, разработанный Google. Ключевые особенности:
- Основан на Protocol Buffers для эффективной и строго типизированной сериализации данных
- Работает поверх HTTP/2
- Кросс-платформенность

### Типы RPC-вызовов в gRPC
- **Unary RPC** – простой вызов, клиент отправляет один запрос и получает один ответ
- **Server streaming RPC** – клиент отправляет один запрос и получает поток сообщений
- **Client streaming RPC** – клиент отправляет поток сообщений и получает один ответ
- **Bidirectional streaming RPC** – обе стороны могут отправлять потоки сообщений

## Выполнение работы

### 1. Настройка окружения
Создание виртуального окружения и установка библиотек `grpcio` и `grpcio-tools`.

### 2. Создание контракта (.proto файл)
Описание сервиса `OrderManager`, метода `GetOrderStatus`, сообщений `OrderRequest` и `OrderResponse` в соответствии с вариантом.

### 3. Генерация кода
Компиляция `.proto` файла с помощью `protoc` для получения Python-стабов.

### 4. Реализация сервера
Создание класса, наследующего сгенерированный базовый класс сервиса, реализация бизнес-логики метода `GetOrderStatus`, запуск gRPC сервера.

### 5. Реализация клиента
Создание канала подключения, создание стаба, выполнение вызова метода `GetOrderStatus`.

### 6. Тестирование
Запуск сервера и клиента, проверка корректности обмена данными.
