## Теоретические основы

### Обнаружение отказов (Failure Detection)
Механизм, позволяющий узлам распределенной системы определять, когда другие узлы перестали функционировать.

### Протокол Gossip (Сплетен)
Децентрализованный вероятностный протокол, работающий по принципу распространения слухов (эпидемический алгоритм). Каждый узел периодически выбирает случайных соседей и обменивается с ними информацией о состоянии кластера.

### Serf
Реализация gossip-протокола от HashiCorp, используемая для обнаружения узлов и сбоев.

### Ключевые параметры
| Параметр | Описание | Влияние |
|----------|----------|--------|
| Gossip Interval | Время между раундами обмена (сек) | ↓ интервал = ↑ скорость, ↑ трафик |
| Gossip Fanout | Количество опрашиваемых соседей | ↑ fanout = ↑ скорость, ↑ трафик |
| Nodes | Общее количество узлов | Влияет на масштаб системы |
| Packet Loss | Процент потерянных пакетов | ↓ надежность при ↑ потерь |

### Конвергенция
Состояние, при котором все живые узлы кластера знают о произошедшем событии (например, об отказе узла).

#### 2.2. SerfSimulator (Gossip)
- Узел выбирает fanout случайных соседей
- Учитывается вероятность потери пакета
- Информация распространяется "вирусно"

#### 2.3. HeartbeatSimulator (Full-mesh)
- Каждый узел пингует всех остальных соседей каждый такт
- Самая быстрая сходимость, но квадратичный рост трафика O(N²)

#### 2.4. PingSimulator (Random Probe)
- Узел пингует одного случайного соседа
- Низкий трафик, но медленное обнаружение отказов
